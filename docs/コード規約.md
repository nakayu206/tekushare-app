## 1. 命名規則

### 基本（Dart標準）

| 種類 | 形式 | 例 |
|---|---|---|
| クラス・enum・型 | UpperCamelCase | `WalkSession`, `SpotCategory` |
| 変数・関数・引数 | lowerCamelCase | `walkSession`, `saveSpot()` |
| 定数 | lowerCamelCase | `defaultTimerMinutes` |
| ファイル名 | snake_case | `walk_session.dart` |
| ディレクトリ名 | snake_case | `walk/`, `common_widgets/` |

### プロジェクト固有ルール

#### Provider の命名

```dart
// ✅ 良い: 〇〇Provider で統一
final walkSessionProvider = ...
final spotsProvider = ...
final filteredSpotsProvider = ...

// ❌ 悪い: 種類がバラバラ
final walkSessionState = ...
final getSpots = ...
```

#### Notifier の命名

```dart
// ✅ 良い: 〇〇Notifier
class WalkSessionNotifier extends StateNotifier<WalkSession> {}
class InactivityNotifier extends StateNotifier<bool> {}
```

#### Entity / Model の命名

```dart
// domain層: シンプルな名前
class Spot {}
class WalkRoute {}

// data層: Modelサフィックスをつける
class SpotModel {}
class WalkRouteModel {}
```

#### UseCase の命名

```dart
// ✅ 動詞 + 名詞 で何をするか明確に
class SaveSpot {}
class UpdateSpotStatus {}
class CheckInactivity {}

// ❌ 曖昧
class SpotUseCase {}
class WalkManager {}
```

#### Widget の命名

```dart
// ✅ 画面: 〇〇Page
class WalkPage extends ConsumerWidget {}

// ✅ 部品: 〇〇Widget または役割名
class TimerWidget extends StatelessWidget {}
class WantToGoButton extends StatelessWidget {}

// ❌ 画面なのにWidgetやScreenが混在
class WalkScreen {}
class MapWidget {} // 画面レベルのもの
```

#### bool 変数・関数

```dart
// ✅ is / has / can で始める
bool isWalking = false;
bool hasPhoto = true;
bool canSaveSpot = false;

// ❌ 状態が不明
bool walk = false;
bool photo = true;
```

## 2. ファイル・クラスの責務範囲

### 1ファイル1クラスを基本とする

```dart
// ✅ spot.dart にはSpotクラスだけ
class Spot {
  final String id;
  final double lat;
  final double lng;
  ...
}

// ❌ 複数クラスを1ファイルに詰め込まない
class Spot { ... }
class SpotFilter { ... }  // 別ファイルに分ける
```

**例外:** 密結合な小さなクラス（enumや小さなValueObject）は同じファイルに置いてよい

```dart
// spot.dart 内にまとめてOK
enum SpotStatus { wantToGo, visited }
enum SpotCategory { cafe, park, shop, restaurant, other }

class Spot { ... }
```

### Widgetの分割基準

**分割する目安:**
- 100行を超えたら分割を検討する
- 同じWidgetが2箇所以上で使われたら `common_widgets/` に移動する
- `build()` のネストが4階層を超えたら部品化する

```dart
// ✅ 役割ごとに分割
// walk_page.dart
class WalkPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    return Scaffold(
      body: Column(
        children: [
          TimerWidget(),       // timer_widget.dart
          WalkMapView(),       // walk_map_view.dart
          WantToGoButton(),    // want_to_go_button.dart
        ],
      ),
    );
  }
}
```

### Page と Widget の責務分離

```dart
// ✅ Pageはデータ取得と画面構成のみ
class SpotListPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final spots = ref.watch(filteredSpotsProvider);
    return SpotListView(spots: spots); // 表示はWidgetに委譲
  }
}

// ✅ Widgetはデータを受け取って表示するだけ
class SpotListView extends StatelessWidget {
  const SpotListView({required this.spots});
  final List<Spot> spots;
  ...
}
```

## 3. コメントルール

### 言語: 日本語で書く

```dart
// ✅ 日本語でOK（チーム内ドキュメントなので）
/// スポットを現在地に保存する
/// [category] を省略した場合は [SpotCategory.other] を使用する
Future<void> saveSpot({SpotCategory? category}) async { ... }

// ❌ 英語・日本語が混在しない
/// Save spot to current location
```

### ドキュメントコメント（`///`）を書く場所

```dart
// ✅ 必ず書く: publicなクラス・関数・プロパティ
/// 散歩セッションの状態を管理するNotifier
/// idle → walking → finished の順で遷移する
class WalkSessionNotifier extends StateNotifier<WalkSession> {

  /// 散歩を開始する。GPS記録とタイマーをスタートさせる
  Future<void> startWalk() async { ... }
}

// ✅ 書かなくていい: privateメソッド・明白な変数
double _lat = 0.0; // コメント不要
```

### インラインコメント（`//`）のルール

```dart
// ✅ 「なぜそうしているか」を書く（whatではなくwhy）
// Isarは同一スレッドでの書き込みしか保証しないため、isolateで実行
await isar.writeTxn(() async { ... });

// ❌ コードを日本語で言い直しているだけ（不要）
// spotsを取得する
final spots = await spotRepository.getAll();
```

## 4. 非同期処理の書き方

### `async/await` を使う（`.then()` は使わない）

```dart
// ✅ async/await
Future<void> saveSpot() async {
  final position = await locationService.getCurrentPosition();
  await spotRepository.save(spot);
}

// ❌ .then() チェーン（可読性が下がる）
locationService.getCurrentPosition().then((position) {
  spotRepository.save(spot).then((_) { ... });
});
```

### エラーハンドリングの統一

```dart
// ✅ try/catch で明示的に処理する
Future<void> saveSpot() async {
  try {
    final position = await locationService.getCurrentPosition();
    await spotRepository.save(spot);
  } on LocationPermissionException {
    // 権限エラーは専用の処理
    state = state.copyWith(error: AppError.locationPermission);
  } catch (e) {
    // 予期しないエラー
    state = state.copyWith(error: AppError.unknown);
  }
}

// ❌ エラーを握りつぶさない
Future<void> saveSpot() async {
  try {
    await spotRepository.save(spot);
  } catch (_) {} // ← 絶対NG
}
```

### Riverpod での非同期状態

```dart
// ✅ AsyncValue を活用してローディング・エラーを表現
final spotsProvider = FutureProvider<List<Spot>>((ref) async {
  return ref.watch(spotRepositoryProvider).getAll();
});

// UI側
final spots = ref.watch(spotsProvider);
return spots.when(
  data: (list) => SpotListView(spots: list),
  loading: () => const LoadingIndicator(),
  error: (e, _) => ErrorView(message: e.toString()),
);
```

### `unawaited` の扱い

```dart
// ✅ 意図的に待たない場合は unawaited() で明示
unawaited(notificationService.scheduleReturnNotification());

// ❌ 暗黙的に await を忘れない
notificationService.scheduleReturnNotification(); // 警告が出る
```

## 5. Riverpodの使い方ルール

### `watch` / `read` の使い分け

```dart
// ✅ watch: 値の変化に応じてUIを再描画したい
final spots = ref.watch(spotsProvider);

// ✅ read: ボタンタップなどイベント時に1回だけ呼ぶ
onPressed: () => ref.read(walkSessionProvider.notifier).startWalk(),

// ❌ build()内でreadを使わない（変化を検知できない）
Widget build(BuildContext context, WidgetRef ref) {
  final spots = ref.read(spotsProvider); // ← NG
}
```

### Provider の定義場所

```dart
// ✅ 関連するファイルのトップレベルに定義
// spot_provider.dart
final spotsProvider = FutureProvider<List<Spot>>((ref) { ... });
final filteredSpotsProvider = Provider<List<Spot>>((ref) { ... });
final spotFilterProvider = StateProvider<SpotFilter>((ref) { ... });

// ❌ クラス内に定義しない
class SpotPage extends ConsumerWidget {
  final provider = Provider(...); // ← NG（毎回再生成される）
}
```

### ConsumerWidget の使い方

```dart
// ✅ 画面・データを参照するWidgetはConsumerWidget
class WalkPage extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) { ... }
}

// ✅ データを受け取るだけのWidgetはStatelessWidget（refは不要）
class TimerDisplay extends StatelessWidget {
  const TimerDisplay({required this.elapsed});
  final Duration elapsed;
}
```

### 状態の更新は必ず Notifier 経由

```dart
// ✅ Notifier のメソッド経由で更新
ref.read(walkSessionProvider.notifier).startWalk();

// ❌ 直接 state を変えない（外部から触れないようにする）
ref.read(walkSessionProvider.notifier).state = WalkSession.walking(...);
```

## 6. importの順序

以下の順番で記述し、グループ間に1行空白を入れる。

```dart
// 1. dart標準ライブラリ
import 'dart:async';
import 'dart:io';

// 2. Flutter
import 'package:flutter/material.dart';
import 'package:flutter/services.dart';

// 3. 外部パッケージ（pub.dev）
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:geolocator/geolocator.dart';
import 'package:isar/isar.dart';

// 4. 内部パッケージ（プロジェクト内）
import 'package:aruki_memo/domain/entities/spot.dart';
import 'package:aruki_memo/presentation/providers/spot_provider.dart';
```

**ルール:**
- 相対パス（`../`）は使わず、必ず `package:aruki_memo/` の絶対パスで書く
- 未使用の import は残さない（lintで自動検出される）

## 7. constの使い方

### 積極的に `const` をつける

```dart
// ✅ コンストラクタ引数が全てコンパイル時定数なら const
const LoadingIndicator()
const SizedBox(height: 16)
const EdgeInsets.all(16)
const Text('行きたい')

// ✅ Widgetのコンストラクタに const をつける
class WantToGoButton extends StatelessWidget {
  const WantToGoButton({super.key}); // ← super.keyも忘れずに
}
```

### `const` をつけられない場合

```dart
// 実行時に値が決まるものはconstにできない（つけなくていい）
Text(spot.name)          // 変数を含む
SizedBox(height: margin) // 変数を含む
DateTime.now()           // 実行時に決まる
```

### `const` と `final` の使い分け

```dart
// const: コンパイル時定数（変更不可・再利用される）
const double defaultTimerMinutes = 15.0;
const Color primaryGreen = Color(0xFF2E7D32);

// final: 実行時に1度だけ代入（変更不可）
final spot = Spot(id: uuid.v4(), ...); // 実行時に生成
final now = DateTime.now();            // 実行時に決まる
```

```yaml
include: package:flutter_lints/flutter.yaml

  rules:
    # constを強制
    prefer_const_constructors: true
    prefer_const_declarations: true
    prefer_const_literals_to_create_immutables: true

    # import順を強制
    directives_ordering: true

    # 未使用コードを検出
    unused_import: true
    unused_local_variable: true

    # その他
    prefer_final_fields: true
    avoid_print: true          # print()の代わりにdebugPrint()を使う
    use_super_parameters: true # super.key を強制
```
