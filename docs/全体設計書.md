## 1. プロジェクト概要

**アプリ名:** てくしぇあ（TekuShare）

**コンセプト**

> 「散歩中に気になった場所を、立ち止まらず1タップで記録。後から地図とリストで振り返れる。」

毎日の散歩中に「この店いいな」「この公園また来たい」と感じた瞬間を、アプリを操作することなく1タップで記録できる。散歩のルートも自動で保存され、行きたい場所・行った場所を地図とリストで管理できる。

| 項目 | 内容 |
|---|---|
| 技術スタック | Flutter / Dart |
| 対応OS | iOS / Android |
| バージョン | v1.0 |

## 3. テーマ・コンセプト

### テーマ

**「日常の散歩を、記録と発見で豊かにする」**

### なぜこのアプリが必要か

既存のアプリでは、散歩に必要な機能が1つに揃っていない。

| アプリ | できること | あるきメモとの違い |
|---|---|---|
| Googleマップ | 場所保存・ルート案内 | 操作が多い・散歩特化でない |
| Strava等 | ルート記録・タイム計測 | スポーツ向け・スポット保存なし |
| メモ帳 | 自由にメモ | 地図と連動しない |
| **あるきメモ** ✅ | **全部を散歩に特化して統合** | ← ここが差別化ポイント |

### ターゲットユーザー

**メインターゲット**
毎日決まった時間に近所を散歩している30〜60代。健康維持を目的としながら、日常の中の小さな発見を楽しんでいる人。スマホは使えるが、複雑な操作は避けたい層。

**サブターゲット**
子連れ・ペット連れで近所を歩く人。公園・カフェ・休憩スポットを記録したいニーズが高く、家族シェア機能と相性が良い。

## 4. 主要機能と「なぜ必要か」

### 📍 行きたいボタン（1タップ即時登録）

- **課題:** 散歩中にGoogleマップを開いて検索して保存する操作が面倒で、結局忘れてしまう
- **解決:** 1タップで現在地をスポットとして即保存。散歩中の摩擦ゼロが最大の価値

### ✅ 行きたい → 行った ステータス管理

- **課題:** 保存しても「どこに行ったっけ」「まだ行ってないのはどこ？」が分からなくなる
- **解決:** 保存したスポットが次の散歩の目的地になり、達成感が継続モチベーションに変わる

### ⏱ 散歩タイマー（15分折返し通知）

- **課題:** 歩きすぎて帰り道がしんどい・時間を気にしながら歩くのがストレス
- **解決:** 15分で折返し通知 → 往復30分が自動完結。習慣化を仕組みでサポートする

### 🗺 ルート可視化

- **課題:** 毎日歩いているけど記録が残らない。同じ道ばかりになりがち
- **解決:** 過去のルートが地図に残ることで、新しい道を開拓したくなる行動変容が生まれる

### 👥 家族・友人とのシェア

- **課題:** おすすめの場所をLINEで伝えても地図情報が伝わりにくい
- **解決:** 共有スポットリストで「今日ここ行ってみよう」が家族間でスムーズに決まる

### 🆘 安否確認（一定時間無動作で通知）

- **課題:** 高齢者が一人で散歩中に体調不良や転倒しても、家族がすぐに気づけない
- **解決:** 散歩中にGPSの動きが一定時間（例: 10分）止まったら家族のスマホに自動通知。「元気です」ボタンで誤通知をキャンセルできる
- **技術メモ:** `geolocator` の位置変化監視 + `flutter_local_notifications` + Phase2のFirebase Cloud Messagingで家族端末に通知

### 📷 写真をマップに紐づける

- **課題:** 気になった場所の雰囲気を言葉だけで残すのは難しい。後から見返しても場所のイメージが思い出せない
- **解決:** その場で撮影 or カメラロールから選んでスポットに写真を追加。地図のピンをタップすると写真も一緒に見られる
- **技術メモ:** `image_picker` パッケージで撮影・選択両対応。写真はローカル保存、Phase2でFirebase Storageに同期

## 5. 技術スタック

| 用途 | パッケージ | 備考 |
|---|---|---|
| メインフレーム | Flutter / Dart | iOS・Android 同時開発 |
| 位置情報 | geolocator | GPS追跡・iOS/Android両対応 |
| 地図表示 | flutter_map + OSM | 完全無料・APIキー不要 |
| ローカルDB | isar | ルート座標配列の保存に強い |
| 状態管理 | riverpod | 業界標準・テスト容易 |
| 折返し通知 | flutter_local_notifications | バックグラウンド通知対応・安否確認にも使用 |
| 距離計算 | latlong2 | 座標間距離の計算・無動作判定にも使用 |
| 写真選択・撮影 | image_picker | 撮影・カメラロール両対応 |
| Phase2: 認証/DB | Firebase Auth + Firestore | 無料枠で個人利用はゼロ円運用可 |
| Phase2: 家族通知 | Firebase Cloud Messaging | 安否確認の家族端末へのプッシュ通知 |
| Phase2: 写真保存 | Firebase Storage | 写真のクラウド同期・シェア対応 |

### コスト設計

```
MVP段階:
  地図 → OpenStreetMap（完全無料）
  DB   → ローカル保存のみ（サーバー代ゼロ）

リリース時:
  Firebase 無料枠（Sparkプラン）
    → Firestore: 1GB保存, 10万読取/日
    → Auth: 無制限
    → 個人利用レベルなら余裕でゼロ円運用可能
```

## 9. エラー・エッジケース定義

### 🔴 高優先

#### GPS が取得できない
- **状況:** 建物内・地下・機内モード時
- **対応:** 「GPS信号を確認中...」を表示 → 30秒取得できなければ「屋外に出てから開始してください」ダイアログ → 散歩開始ボタンを無効化
- **スポット保存:** GPS取得中は「行きたい」ボタンをグレーアウト

#### 位置情報の権限がない
- **状況:** 初回起動・権限拒否後
- **対応:** 初回は「位置情報の許可が必要です」説明画面 → 許可ダイアログ → 拒否された場合は設定アプリへ誘導するボタンを表示
- **注:** 権限なしでもリスト・履歴は閲覧可能

#### 散歩中に強制終了・クラッシュ
- **状況:** OSによる強制終了・バッテリー切れ
- **対応:** GPS座標を30秒ごとにIsarに自動保存（中間保存）→ 再起動時に「中断したルートがあります。続きから再開しますか？」を表示
- **スポット:** 保存ボタンを押した時点で即時コミット

---

### 🟡 中優先

#### 散歩中にアプリがバックグラウンドに
- **状況:** 電話・通知・ホームボタンで離脱
- **対応:** タイマーは継続（バックグラウンドで計測）。GPS記録は一時停止 → 復帰時に「X分間記録が途切れました」をトーストで通知
- **折返し通知:** バックグラウンドでも通知は届く（flutter_local_notifications）

#### 写真サイズが大きすぎる
- **状況:** 4K写真・RAW等（10MB超）
- **対応:** image_picker で取得後に自動リサイズ（最大1200px / 80%品質）→ ローカル保存。ユーザーに通知不要
- **Phase2:** Firebase Storage アップロード前にも同様の圧縮を実施

#### 安否確認の誤通知
- **状況:** ベンチで休憩中・立ち話中
- **対応:** 通知前に「元気です」確認ダイアログを30秒間表示 → タップで通知キャンセル → 無反応の場合のみ家族通知（Phase2）
- **設定:** 無動作検知時間は設定画面で5〜30分に変更可能

---

### ⚪ 低優先

#### ストレージ不足
- **状況:** 長期利用でデータが蓄積
- **対応:** 設定画面に「データ使用量」表示 → 古いルートの座標を自動間引き（30日以上経過 + 500点超のルート）→ 手動削除オプションも提供

## 10. 状態管理設計（Riverpod）

### 層の考え方

```
Infrastructure → Repository → State → Derived → UI
（OS/HW）       （DB/API）   （状態）  （派生）   （画面）
```

### Infrastructure層（ServiceProvider）

```dart
// GPS ストリームを管理
final locationServiceProvider = Provider((ref) => LocationService());

// 通知（折返し・安否確認）を管理
final notificationServiceProvider = Provider((ref) => NotificationService());

// カメラ・カメラロールを管理
final cameraServiceProvider = Provider((ref) => CameraService());
```

### Repository層

```dart
// スポットのCRUD
final spotRepositoryProvider = Provider((ref) {
  return SpotRepositoryImpl(isar: ref.watch(isarProvider));
});

// ルートのCRUD
final routeRepositoryProvider = Provider((ref) {
  return RouteRepositoryImpl(isar: ref.watch(isarProvider));
});

// 写真の保存・取得
final photoRepositoryProvider = Provider((ref) {
  return PhotoRepositoryImpl();
});
```

### State層（核心）

```dart
// 散歩セッション全体の状態（一番複雑）
// 状態: idle | walking | finished
final walkSessionProvider =
    StateNotifierProvider<WalkSessionNotifier, WalkSession>((ref) {
  return WalkSessionNotifier(
    locationService: ref.watch(locationServiceProvider),
    routeRepo:       ref.watch(routeRepositoryProvider),
    notifService:    ref.watch(notificationServiceProvider),
  );
});

// GPS座標をリアルタイムでStream
final locationProvider = StreamProvider<LatLng>((ref) {
  return ref.watch(locationServiceProvider).positionStream;
});

// 保存済みスポット一覧
final spotsProvider = FutureProvider<List<Spot>>((ref) {
  return ref.watch(spotRepositoryProvider).getAll();
});

// 散歩履歴一覧
final walkHistoryProvider = FutureProvider<List<WalkRoute>>((ref) {
  return ref.watch(routeRepositoryProvider).getAll();
});
```

### Derived層（派生・フィルタ）

```dart
// フィルタ条件（カテゴリ・ステータス）の状態
final spotFilterProvider = StateProvider((ref) => SpotFilter.all());

// フィルタを適用したスポット一覧
final filteredSpotsProvider = Provider<List<Spot>>((ref) {
  final spots  = ref.watch(spotsProvider).valueOrNull ?? [];
  final filter = ref.watch(spotFilterProvider);
  return filter.apply(spots);
});

// 散歩の統計（距離・時間）を計算
final walkStatsProvider = Provider<WalkStats>((ref) {
  final session = ref.watch(walkSessionProvider);
  return WalkStats.from(session);
});

// 無動作検知（安否確認）
final inactivityProvider = StateNotifierProvider<InactivityNotifier, bool>((ref) {
  return InactivityNotifier(
    locationStream: ref.watch(locationProvider.stream),
    notifService:   ref.watch(notificationServiceProvider),
  );
});
```

### 各画面が参照するプロバイダー

| 画面 | 参照するプロバイダー |
|---|---|
| 散歩モード | `walkSessionProvider` `locationProvider` `walkStatsProvider` `inactivityProvider` |
| 地図画面 | `spotsProvider` `walkHistoryProvider` `spotFilterProvider` |
| リスト画面 | `filteredSpotsProvider` `spotFilterProvider` |
| 履歴画面 | `walkHistoryProvider` |
| 設定画面 | `walkSessionProvider`（タイマー設定） |

### WalkSession の状態遷移

```
idle
  └─[開始ボタン]→ walking
                    ├─[15分経過]→ 折返し通知（stateはwalkingのまま）
                    ├─[10分無動作]→ 安否確認通知（stateはwalkingのまま）
                    └─[終了ボタン]→ finished
                                      └─[保存完了 or 破棄]→ idle
```

### 各層の役割

| 層 | 役割 | 依存していいもの |
|---|---|---|
| `domain` | ビジネスロジックの核。FlutterもIsarも知らない | 何にも依存しない |
| `data` | domainのインターフェースを実装。Isar/Firebaseを知る | domain のみ |
| `presentation` | UIとRiverpodプロバイダー | domain・data |
| `infrastructure` | GPS・通知・カメラのOS依存処理 | なし（serviceとして独立） |
| `core` | 全層から使う定数・ユーティリティ | なし |

### Phase2追加時の変更範囲

```
変更が必要な場所だけ:
  data/datasources/remote/ → Firestoreの実装を追加
  data/repositories/*_impl → ローカル/リモートを切り替えるロジック追加
  infrastructure/          → FCM（プッシュ通知）サービス追加

domain層・presentation層は一切触らなくていい ← クリーンアーキテクチャの恩恵
```
