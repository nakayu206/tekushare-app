# TekuShare 環境構築手順

> 新しいPC・新メンバー参加時の初期セットアップ手順。
> 上から順に実行すれば動かせる状態になる。

---

## 0. 前提

| 項目 | バージョン |
|---|---|
| OS | Windows 10/11 または macOS |
| Flutter | 3.41.x 以上 |
| Dart | 3.11.x 以上（Flutter同梱） |
| Android Studio | 最新安定版 |
| Xcode | 最新安定版（macOSのみ） |
| VS Code | 最新版（推奨エディタ） |

---

## 1. Flutter SDK のインストール

### Windows

1. [Flutter SDK](https://docs.flutter.dev/get-started/install/windows) をダウンロード
2. `C:\src\flutter` などに展開（パスにスペース・日本語は禁止）
3. 環境変数 `Path` に `C:\src\flutter\bin` を追加
4. PowerShell で確認:
   ```powershell
   flutter --version
   ```

### macOS

```bash
# Homebrew 経由
brew install --cask flutter

# 確認
flutter --version
```

---

## 2. Android 開発環境

### 2-1. Android Studio をインストール

[Android Studio 公式](https://developer.android.com/studio) からインストーラーを取得して実行。
初回起動時のセットアップウィザードで以下を入れる:

- Android SDK
- Android SDK Platform
- Android Virtual Device

### 2-2. ライセンス同意

```powershell
# Windows
$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
flutter doctor --android-licenses
```

```bash
# macOS
flutter doctor --android-licenses
```

すべて `y` で同意する。

### 2-3. エミュレータ作成（AVD）

#### コマンドラインで作成する場合（Windows）

```powershell
$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"

# システムイメージのインストール（Android 14 / Google Play / x86_64）
'y' | & "$env:LOCALAPPDATA\Android\Sdk\cmdline-tools\latest\bin\sdkmanager.bat" `
  "system-images;android-34;google_apis_playstore;x86_64"

# AVD 作成（pixel_6 ベース）
echo "no" | & "$env:LOCALAPPDATA\Android\Sdk\cmdline-tools\latest\bin\avdmanager.bat" `
  create avd -n tekushare_pixel `
  -k "system-images;android-34;google_apis_playstore;x86_64" `
  -d pixel_6
```

#### Android Studio で作成する場合

`More Actions` → `Virtual Device Manager` → `Create Device` で Pixel 6 / API 34 を選択。

## 4. VS Code セットアップ

### 4-1. VS Code 本体

[VS Code 公式](https://code.visualstudio.com/) からインストール。

### 4-2. 必須拡張機能

| 拡張 | 用途 |
|---|---|
| Flutter | デバッグ・補完・ホットリロード |
| Dart | Flutter に同時インストールされる |
| GitHub Copilot | コード補助（任意） |

VS Code のターミナルで:

```powershell
code --install-extension Dart-Code.flutter
```

### 4-3. 推奨設定

`.vscode/settings.json`（リポジトリ内に作る場合）:

```json
{
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll": "explicit",
    "source.organizeImports": "explicit"
  },
  "[dart]": {
    "editor.rulers": [80],
    "editor.selectionHighlight": false,
    "editor.suggestSelection": "first",
    "editor.tabCompletion": "onlySnippets"
  }
}
```

## 5. プロジェクトのセットアップ

### 5-1. リポジトリをクローン

```powershell
git clone https://github.com/nakayu206/tekushare-app.git
cd tekushare-app
```

### 5-2. 依存パッケージ取得

```powershell
flutter pub get
```

### 5-3. 環境チェック

```powershell
flutter doctor -v
```

すべて ✅（√） であれば OK。Android toolchain や Xcode が ❌ の場合は対応する章へ戻る。

## 6. 動作確認

### 6-1. エミュレータ起動

```powershell
# Windows
$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
& "$env:LOCALAPPDATA\Android\Sdk\emulator\emulator.exe" -avd tekushare_pixel
```

または VS Code 右下のデバイスステータスバーから選択。

### 6-2. デバイス確認

```powershell
flutter devices
```

エミュレータが `emulator-5554` として表示されれば OK。

### 6-3. アプリ起動

| 環境 | コマンド |
|---|---|
| dev | `flutter run --flavor dev -t lib/main_dev.dart` |
| stg | `flutter run --flavor stg -t lib/main_stg.dart` |
| prod | `flutter run --flavor prod -t lib/main_prod.dart --release` |

VS Code 派は `F5` → 構成（dev/stg/prod）を選択でOK。

### 6-4. テスト・解析

```powershell
flutter analyze   # Lint チェック
flutter test      # ユニットテスト
```

## 7. 環境（Flavor）の使い分け

| Flavor | applicationId | アプリ名 | 用途 |
|---|---|---|---|
| dev | `com.example.tekushare.dev` | TekuShare Dev | 開発・検証 |
| stg | `com.example.tekushare.stg` | TekuShare Stg | テスト・QA |
| prod | `com.example.tekushare` | TekuShare | リリース |

3 環境とも別 applicationId なので **同じ端末に並行インストール可能**。

## 8. リリースAPK / IPA のビルド

### Android

```powershell
# AAB（Google Play 推奨）
flutter build appbundle --flavor prod -t lib/main_prod.dart --release

# APK
flutter build apk --flavor prod -t lib/main_prod.dart --release
```

成果物:
- `build/app/outputs/bundle/prodRelease/app-prod-release.aab`
- `build/app/outputs/flutter-apk/app-prod-release.apk`

> ⚠️ 本番リリース前に署名設定が必要（[Flutter公式](https://docs.flutter.dev/deployment/android#signing-the-app)）。

### iOS（macOS のみ）

```bash
flutter build ios --flavor prod -t lib/main_prod.dart --release
```

詳細は [`ios/FLAVOR_SETUP.md`](../ios/FLAVOR_SETUP.md) 参照。

## 9. トラブルシューティング

### ❌ `JAVA_HOME` が設定されていない

```powershell
$env:JAVA_HOME = "C:\Program Files\Android\Android Studio\jbr"
```

PowerShell プロファイル（`$PROFILE`）に書いておくと毎回設定不要。

### ❌ エミュレータが起動しない（lock ファイル残存）

```powershell
Get-Process emulator,qemu-system-* -ErrorAction SilentlyContinue | Stop-Process -Force
Remove-Item -Recurse -Force "$env:USERPROFILE\.android\avd\tekushare_pixel.avd\*.lock"
```

### ❌ `flutter pub get` が遅い・失敗する

```powershell
flutter clean
flutter pub get
```

それでも直らない場合は `pubspec.lock` を削除して再取得。

### ❌ Gradle ビルドが遅い

初回のみ 2〜3分かかる（依存ダウンロード）。2回目以降はキャッシュで早くなる。

### ❌ `flutter analyze` でエラー

```powershell
dart fix --apply       # 自動修正
dart format .          # フォーマット
```
